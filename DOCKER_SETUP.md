# Docker & Database Setup Guide

## Quick Start (Local Development with Docker Compose)

### Prerequisites
- Docker Desktop installed
- Docker Compose installed

### 1. Start Everything with One Command
```bash
docker-compose up --build
```

This will:
- ✅ Build the Spring Boot application
- ✅ Start MySQL database
- ✅ Wait for MySQL to be healthy
- ✅ Start the Spring Boot app
- ✅ Create the `college_events` database
- ✅ Application available at `http://localhost:8080`

### 2. Stop Everything
```bash
docker-compose down
```

To also remove volumes (clears database):
```bash
docker-compose down -v
```

## Setup Requirements

### Environment Variables Setup

**Create a `.env` file in the project root:**
```bash
copy .env.example .env
```

**Edit `.env` with your values:**
```properties
# Local MySQL
DB_HOST=mysql
DB_PORT=3306
DB_NAME=college_events
DB_USERNAME=eventra_user
DB_PASSWORD=eventra_password

# JWT
JWT_SECRET=your-secret-key-here

# Gmail SMTP
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password
MAIL_FROM=your-email@gmail.com

# Spring Profile
SPRING_PROFILES_ACTIVE=dev
```

### Gmail Setup for Email Notifications

1. Go to https://myaccount.google.com/apppasswords
2. Select "Mail" and "Windows Computer"
3. Generate 16-character App Password
4. Use it in `.env` as `MAIL_PASSWORD`

## Running the Application

### Option 1: Docker Compose (Recommended for Local Development)

```bash
# Build and start all services
docker-compose up --build

# Run in background
docker-compose up -d --build

# View logs
docker-compose logs -f app

# View database logs
docker-compose logs -f mysql
```

### Option 2: Docker Only (Requires External MySQL)

```bash
# Build Docker image
docker build -t eventra-backend:latest .

# Run with environment variables
docker run -p 8080:8080 \
  -e DB_HOST=<your-mysql-host> \
  -e DB_PORT=3306 \
  -e DB_NAME=college_events \
  -e DB_USERNAME=root \
  -e DB_PASSWORD=pass123 \
  -e JWT_SECRET=your-secret \
  -e SPRING_PROFILES_ACTIVE=prod \
  eventra-backend:latest
```

### Option 3: Local Maven (Development)

```bash
# Requirements: MySQL running on localhost:3306

# Build
mvn clean install

# Run with dev profile
mvn spring-boot:run -Dspring-boot.run.arguments="--spring.profiles.active=dev"

# Or run the JAR
java -jar target/event-management-system-1.0.0.jar
```

## Database Setup

### Automatic Setup (Docker Compose)
- MySQL automatically created
- Database `college_events` created
- Tables auto-generated by Hibernate

### Manual Setup (Local MySQL)

```bash
# Start MySQL
mysql -u root -p

# Create database
CREATE DATABASE college_events;

# Create user
CREATE USER 'eventra_user'@'%' IDENTIFIED BY 'eventra_password';
GRANT ALL PRIVILEGES ON college_events.* TO 'eventra_user'@'%';
FLUSH PRIVILEGES;
```

### Deploy to Render.com

```bash
# 1. Create render.yaml in project root
# 2. Push to GitHub
# 3. Connect GitHub repo to Render
# 4. Set environment variables in Render dashboard:
#    - DB_HOST
#    - DB_PORT
#    - DB_NAME
#    - DB_USERNAME
#    - DB_PASSWORD
#    - JWT_SECRET
#    - MAIL_USERNAME
#    - MAIL_PASSWORD
# 5. Deploy
```

### Deploy to Azure Container Instances

```bash
# Build and push to Azure Container Registry
az acr build --registry <registry-name> --image eventra-backend:latest .

# Create container instance
az container create \
  --resource-group <resource-group> \
  --name eventra-backend \
  --image <registry-name>.azurecr.io/eventra-backend:latest \
  --cpu 1 --memory 1.5 \
  --environment-variables \
    DB_HOST=<your-mysql-server> \
    DB_USERNAME=<username> \
    DB_PASSWORD=<password> \
    JWT_SECRET=<your-secret> \
  --ports 8080
```

### Deploy to AWS ECS

```bash
# Push to ECR
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin <account-id>.dkr.ecr.us-east-1.amazonaws.com

docker tag eventra-backend:latest <account-id>.dkr.ecr.us-east-1.amazonaws.com/eventra-backend:latest
docker push <account-id>.dkr.ecr.us-east-1.amazonaws.com/eventra-backend:latest

# Create ECS task definition with environment variables
```

## Troubleshooting

### MySQL Connection Error
```
Error: Communications link failure
```

**Solutions:**
1. Ensure MySQL container is running: `docker-compose ps`
2. Check MySQL is healthy: `docker-compose logs mysql`
3. Wait 10-15 seconds for MySQL to start
4. Verify DB_HOST is correct (use `mysql` for compose, not `localhost`)

### Port Already in Use
```
Error: Address already in use: JVM_Bind
```

**Solutions:**
```bash
# Change port in docker-compose.yml or
# Kill process on port 8080
lsof -ti:8080 | xargs kill -9

# Or use different port
docker run -p 9090:8080 eventra-backend:latest
```

### Database Migration Failed
```
Error: Unable to build Hibernate SessionFactory
```

**Solutions:**
1. Remove volumes and restart: `docker-compose down -v && docker-compose up`
2. Check `application-prod.properties` DDL setting
3. Ensure database user has all privileges
4. Check database character set: `DEFAULT CHARSET=utf8mb4`

### Environment Variables Not Picked Up
```bash
# Restart container with new env vars
docker-compose restart

# Or rebuild
docker-compose down && docker-compose up --build
```

## Performance Tuning

### Database Connection Pool
Adjust in `application-prod.properties`:
```properties
spring.datasource.hikari.maximum-pool-size=20
spring.datasource.hikari.minimum-idle=10
```

### Hibernate Batch Settings
```properties
spring.jpa.properties.hibernate.jdbc.batch_size=50
spring.jpa.properties.hibernate.order_inserts=true
spring.jpa.properties.hibernate.order_updates=true
```

### Memory Configuration
```bash
# Run with more memory
docker run -e JAVA_OPTS="-Xmx512m -Xms256m" eventra-backend:latest
```

## Maintenance

### Database Backup
```bash
# Backup database
docker-compose exec mysql mysqldump -u root -proot college_events > backup.sql

# Restore database
docker-compose exec -T mysql mysql -u root -proot college_events < backup.sql
```

### Clear Application Cache
```bash
docker-compose restart app
```

### Update Dependencies
```bash
# Update Maven dependencies
mvn dependency:update

# Rebuild Docker image
docker-compose build --no-cache app
```

# RENDER DEPLOYMENT - STEP-BY-STEP FINAL FIX

## Current Problem

Your Render deployment is failing with:
```
java.net.ConnectException: Connection refused
Unable to open JDBC Connection for DDL execution
```

**Why:** The web service environment variables are NOT set, so the app defaults to `localhost:3306`.

---

## ⏱️ EXACT STEPS (Takes 10 minutes)

### Step 1: Verify MySQL Database Exists (2 minutes)

1. Go to https://render.com/dashboard
2. Look at the left sidebar under **Resources**
3. Find resource named `college-events-db` (should be a MySQL database)
4. Check for **green checkmark** indicating it's "Available"

**If you see it (green checkmark):**
- ✅ Continue to Step 2

**If you don't see it:**
- ❌ Create it: Click `New +` → `Database` → `MySQL`
- Set values and wait 5 minutes for it to be ready
- Then continue to Step 2

---

### Step 2: Get Database Credentials (1 minute)

1. In Resources list, click on `college-events-db`
2. Scroll down to find the connection string section
3. Copy ONLY the hostname part (should look like: `dpg-xxxx.mysql.render.com`)
4. Also note the password (auto-generated, shown in the dashboard)

---

### Step 3: Set Web Service Environment Variables (3 minutes)

1. In Render Dashboard, go to **Services** (left sidebar)
2. Click on `eventra-backend` (your web service)
3. Click **Settings** (top menu bar)
4. Go to **Environment** section
5. Add/Update these variables:

```
SPRING_PROFILES_ACTIVE
prod

DB_HOST
dpg-xxxx.mysql.render.com

DB_PORT
3306

DB_NAME
college_events

DB_USERNAME
admin

DB_PASSWORD
[copy from step 2]

DB_USE_SSL
true

DDL_AUTO
validate

JWT_SECRET
[generate random 32-char string - can use any secure string]

MAIL_HOST
smtp.gmail.com

MAIL_PORT
587

MAIL_USERNAME
your-email@gmail.com

MAIL_PASSWORD
your-gmail-app-password

MAIL_FROM
your-email@gmail.com
```

**CRITICAL:** After entering each variable, press Enter/Tab to confirm

**Then click "Save" button**

---

### Step 4: Manual Deploy (2 minutes)

1. In Web Service, look for **Deploys** section (top menu)
2. Click **Manual Deploy**
3. Select branch: `main`
4. Click **Deploy**

---

### Step 5: Watch Logs (2 minutes)

1. Click **Logs** tab (top menu)
2. Scroll down to see new logs
3. Watch for either:

**✅ SUCCESS indicators:**
```

Tomcat started on port(s): 8080

```

**❌ FAILURE indicators:**

## ❌ Troubleshooting (If Still Failing)

### Issue: "Connection refused" after setting env vars

**Fix:**
1. Go to Settings → Environment again
2. **Delete** all env vars
3. Add them again one by one (copy-paste carefully)
4. Make sure `DB_HOST` has no `mysql://` prefix, just `dpg-xxxx.mysql.render.com`
5. Save
6. Manual Deploy again

### Issue: "Connection refused" even with correct env vars

**Fix:**
1. Test database directly from your computer:
   ```bash
   mysql -h dpg-xxxx.mysql.render.com -u admin -p
   # When prompted, enter the password from step 2
   ```
   - If this works → database is accessible
   - If fails → database not ready yet, wait 5 more minutes

2. Go to Render Web Service → Settings
3. Scroll to bottom, click **"Clear Build Cache"**
4. Go to Deploys → Manual Deploy → Deploy
5. Wait 10 minutes and check logs again

### Issue: Still failing after clearing cache

**Nuclear option (start fresh):**
1. Go to Web Service settings
2. Scroll to bottom → Click **"Delete Service"**
3. Confirm deletion
4. Wait 2 minutes
5. Create new Web Service:
   - Click `New +` → `Web Service`
   - Connect GitHub again
   - Set env vars again (from Step 3)
   - Deploy

## What Each Env Var Does

| Variable | Purpose | Example |
|----------|---------|---------|
| `SPRING_PROFILES_ACTIVE` | Use production configuration | `prod` |
| `DB_HOST` | MySQL server address | `dpg-abc.mysql.render.com` |
| `DB_PORT` | MySQL port (always 3306) | `3306` |
| `DB_NAME` | Database name | `college_events` |
| `DB_USERNAME` | MySQL username | `admin` |
| `DB_PASSWORD` | MySQL password | Auto-generated by Render |
| `DB_USE_SSL` | Use encrypted connection | `true` |
| `DDL_AUTO` | Hibernate schema mode | `validate` (safe) |
| `JWT_SECRET` | Secret for tokens | Any random string |
| `MAIL_*` | Email configuration | Gmail SMTP settings |
